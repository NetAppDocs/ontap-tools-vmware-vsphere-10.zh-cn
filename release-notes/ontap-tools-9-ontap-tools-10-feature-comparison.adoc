= ONTAP tools for VMware vSphere 9 和 10 功能比较
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
了解从ONTAP tools for VMware vSphere迁移到ONTAP tools for VMware vSphere是否适合您。


NOTE: 有关最新的兼容性信息，请参阅 https://mysupport.netapp.com/matrix["NetApp 互操作性表工具"^]。

[cols="20%,25%,30%"]
|===
| 功能 | ONTAP工具 9.13 | ONTAP工具 10.2 及更高版本 


| 关键价值主张 | 通过增强的安全性、合规性和自动化功能，简化从第 0 天到第 2 天的运营 | 扩展支持，包括 FC for VMFS 和 NVMe-oF for VMFS。NetApp SnapMirror易于使用，vSphere Metro 存储集群设置简单，并支持三站点 VMware Live Site Recovery 


| ONTAP发布资格 | ONTAP 9.9.1 升级至ONTAP 9.16.1 | ONTAP 9.12.1 至 9.15.1 适用于ONTAP工具 10.2。适用于ONTAP工具 10.3 的ONTAP 9.14.1、9.15.1、9.16.0 和 9.16.1。适用于ONTAP工具 10.4 的ONTAP 9.14.1、9.15.1、9.16.0 和 9.16.1。使用ASA r2 系统时， ONTAP工具 10.4 需要ONTAP 9.16.1P3 及更高版本。适用于ONTAP工具 10.5 的ONTAP 9.15.1、9.16.0、9.16.1、9.17.0 和 9.17.1 


| VMware 版本支持 | vSphere 7.x-8.x VMware Site Recovery Manager (SRM) 8.5 至 VMware Live Site Recovery 9.0 | vSphere 7.x-8.x vSphere 9.0（从ONTAP tools 10.5 开始） VMware Site Recovery Manager (SRM) 8.7 至 VMware Live Site Recovery 9.0 *注意：* 在ONTAP tools 10.x 中，SRM 支持共享站点，从而增强了可扩展性并提高了性能。 


| 协议支持 | NFS 和 VMFS 数据存储：NFS（v3 和 v4.1）、VMFS（iSCSI 和 FCP） | NFS 和 VMFS 数据存储：NFS（v3 和 v4.1）、VMFS（iSCSI/FCP/NVMe-oF） 


| 可扩展性 | 主机和虚拟机：300 台主机，最多 1 万台虚拟机；数据存储：600 个 NFS 存储，最多 50 个 VMFS 存储。 | 主机和虚拟机：600 台主机 


| 可观察性 | 性能、容量和主机合规性仪表板动态虚拟机和数据存储区报告 | 更新了性能、容量和主机合规性仪表板动态虚拟机和数据存储区报告 


| 数据保护 | SRA 复制适用于 VMFS 和 NFS。  SCV集成和互操作性，可用于备份。 | 针对 iSCSI VMFS 和 NFS v3 数据存储的 SRA 复制结合了 SMAS 和 VMware Live Site Recovery 的三站点保护。SRA 支持 FCP 与 VMFS。 


| VASA 提供程序支持 | VASA 4.0 | VASA 3.0 
|===
